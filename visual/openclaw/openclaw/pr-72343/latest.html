PR 72343: fresh installs start classic onboarding

ELI5: this PR changes what happens when someone types openclaw for the first time. If OpenClaw has no real config yet, it starts the classic setup wizard. If OpenClaw already has settings, it keeps opening Crestodian. Invalid configs still avoid onboarding so repair remains reachable.

Maintainer Summary

What changes?

Bare openclaw checks the active config. Missing, empty, or metadata-only config means “fresh install,” so it routes to classic onboarding.

What stays the same?

Configured installs still route bare openclaw to Crestodian. Invalid configs do not start onboarding, preserving the repair path.

Main review choice

Maintainers need to decide whether classic onboarding should be the fresh-install default for bare openclaw.
